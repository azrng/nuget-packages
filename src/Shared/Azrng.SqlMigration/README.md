## Azrng.SqlMigration

一个基于 SQL 脚本的数据库迁移 NuGet 包，目前支持 PostgreSQL。

应用启动时会自动扫描指定目录下的版本脚本，按版本顺序执行升级，并将已执行的版本写入版本日志表。

### 快速上手

将需要执行的脚本放到指定目录，例如 `wwwroot/MigrationSql`，脚本文件名使用版本号命名，例如：

* `1.0.0.sql`
* `1.1.0.txt`
* `1.0.0.1.sql`

基础示例：

```csharp
builder.Services.AddSqlMigrationService("default", config =>
{
    config.Schema = "aa";
    config.VersionPrefix = string.Empty;
    config.SqlRootPath = Path.Combine(builder.Environment.WebRootPath, "MigrationSql");
    config.ConnectionBuilder = sp => new NpgsqlConnection(conn);

    // config.LockProvider = x => x.GetRequiredService<ILockProvider>()
    //     .LockAsync("project_init", TimeSpan.FromMinutes(1));
}).AddAutoMigration();
```

### 功能说明

#### 支持的版本号格式

支持以下格式：

* `1.0.0`
* `1.2.3`
* `1.0.0.1`
* `111.111.111`
* `111.111.111.111`

支持的脚本扩展名：

* `.sql`
* `.txt`

#### 多数据库迁移

可以为不同数据库注册多个迁移名称：

```csharp
builder.Services.AddSqlMigrationService("default", config =>
{
    config.Schema = "aa";
    config.VersionPrefix = string.Empty;
    config.SqlRootPath = Path.Combine(builder.Environment.WebRootPath, "MigrationSql");
    config.ConnectionBuilder = sp => new NpgsqlConnection(conn);
}).AddSqlMigrationService("default2", config =>
{
    config.Schema = "bb";
    config.VersionPrefix = string.Empty;
    config.SqlRootPath = Path.Combine(builder.Environment.WebRootPath, "MigrationSql2");
    config.ConnectionBuilder = sp => new NpgsqlConnection(conn2);
}).AddAutoMigration();
```

#### 版本日志表自定义

从 `0.5.0` 开始，支持自定义版本日志表名、主键列、版本列和排序列。

```csharp
builder.Services.AddSqlMigrationService("default", config =>
{
    config.Schema = "aa";
    config.SqlRootPath = Path.Combine(builder.Environment.WebRootPath, "MigrationSql");
    config.ConnectionBuilder = sp => new NpgsqlConnection(conn);

    config.VersionLog.TableName = "custom_version_log";
    config.VersionLog.IdColumn = "log_id";
    config.VersionLog.VersionColumn = "db_version";
    config.VersionLog.OrderByColumn = "log_id";
});
```

说明：

* `OrderByColumn` 未设置时默认跟随 `IdColumn`
* 默认创建的新版本日志表不再包含 `created_time`
* 如果检测到旧版本日志表仍包含 `created_time` 列，会自动兼容写入
* 默认情况下，库会按 `OrderByColumn desc limit 1` 获取当前版本

如果 `OrderByColumn` 既不是 `IdColumn`，也不是 `VersionColumn`，则需要自行提供建表 SQL：

```csharp
config.VersionLog.TableName = "custom_version_log";
config.VersionLog.IdColumn = "log_id";
config.VersionLog.VersionColumn = "db_version";
config.VersionLog.OrderByColumn = "sort_no";
config.VersionLog.InitTableSql = """
CREATE TABLE IF NOT EXISTS "aa"."custom_version_log" (
    "log_id" bigint GENERATED BY DEFAULT AS IDENTITY,
    "db_version" text NOT NULL,
    "sort_no" bigint NOT NULL,
    CONSTRAINT "pk_custom_version_log" PRIMARY KEY ("log_id")
);
CREATE UNIQUE INDEX IF NOT EXISTS "ix_custom_version_log_db_version"
ON "aa"."custom_version_log" ("db_version");
""";
```

#### 初始化版本设置器

适用于中途接入迁移能力的项目。如果数据库里还没有版本日志表，或者版本日志表为空，可以通过初始化版本设置器告诉库“当前数据库已经处于哪个版本”。

```csharp
public class CustomInitVersionSetter : IInitVersionSetter
{
    public Task<string> GetCurrentVersionAsync()
    {
        return Task.FromResult("1.5.0");
    }
}
```

注册方式：

```csharp
builder.Services.AddSqlMigrationService("default", config =>
{
    config.Schema = "aa";
    config.SqlRootPath = Path.Combine(builder.Environment.WebRootPath, "MigrationSql");
    config.ConnectionBuilder = sp => new NpgsqlConnection(conn);
    config.SetInitVersionSetter<CustomInitVersionSetter>();
}).AddAutoMigration();
```

#### 迁移回调

如果需要在迁移前后执行自定义逻辑，可以实现 `IMigrationHandler`：

```csharp
public class DefaultMigrationHandler : IMigrationHandler
{
    private readonly ILogger<DefaultMigrationHandler> _logger;

    public DefaultMigrationHandler(ILogger<DefaultMigrationHandler> logger)
    {
        _logger = logger;
    }

    public Task<bool> BeforeMigrateAsync(string oldVersion)
    {
        _logger.LogInformation("原始版本：{OldVersion}", oldVersion);
        return Task.FromResult(true);
    }

    public Task<bool> VersionUpdateBeforeMigrateAsync(string version)
    {
        _logger.LogInformation("版本 {Version} 开始迁移", version);
        return Task.FromResult(true);
    }

    public Task VersionUpdateMigratedAsync(string version)
    {
        _logger.LogInformation("版本 {Version} 迁移成功", version);
        return Task.CompletedTask;
    }

    public Task VersionUpdateMigrateFailedAsync(string version)
    {
        _logger.LogInformation("版本 {Version} 迁移失败", version);
        return Task.CompletedTask;
    }

    public Task MigratedAsync(string oldVersion, string version)
    {
        _logger.LogInformation("迁移成功，原版本：{OldVersion}，当前版本：{Version}", oldVersion, version);
        return Task.CompletedTask;
    }

    public Task MigrateFailedAsync(string oldVersion, string version)
    {
        _logger.LogInformation("迁移失败，原版本：{OldVersion}，当前版本：{Version}", oldVersion, version);
        return Task.CompletedTask;
    }
}
```

注册方式：

```csharp
builder.Services.AddSqlMigrationService<DefaultMigrationHandler>("default", config =>
{
    config.Schema = "aa";
    config.VersionPrefix = string.Empty;
    config.SqlRootPath = Path.Combine(builder.Environment.WebRootPath, "MigrationSql");
    config.ConnectionBuilder = sp => new NpgsqlConnection(conn);
}).AddAutoMigration();
```

#### 分布式锁处理

为了防止多实例的情况下迁移出现问题，这里可以使用nuget包Azrng.DistributeLock.Redis来实现多实例的情况下只有一个实例进行迁移

```csharp
builder.Services.AddRedisLockProvider("localhost:6379,password=123456,defaultdatabase=0,abortConnect=false");

builder.Services.AddSqlMigrationService("default", config =>
{
    config.Schema = "aa";
    config.VersionPrefix = string.Empty;
    config.SqlRootPath = Path.Combine(builder.Environment.WebRootPath, "MigrationSql");
    config.ConnectionBuilder = sp => new NpgsqlConnection(conn);
    config.LockProvider = x => x.GetRequiredService<ILockProvider>()
        .LockAsync("project_init", TimeSpan.FromMinutes(1));
}).AddAutoMigration();
```

### 注意事项

* 脚本版本号必须递增，否则不会被识别为待执行脚本
* 默认只会扫描当前目录下以 `VersionPrefix` 开头、后缀为 `.sql` 或 `.txt` 的文件
* 每个版本脚本会在独立事务中执行，脚本执行和版本日志写入会在同一个事务中提交
* 如果自定义了 `OrderByColumn`，请确保该列的排序结果能够稳定表示“最新版本”
