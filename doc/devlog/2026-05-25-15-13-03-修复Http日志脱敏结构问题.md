# 修复 Http 日志脱敏结构问题

## 本次目标

- 修复 `Common.HttpClients` 审计日志脱敏后可能破坏 JSON 格式的问题。
- 支持调用方替换默认日志脱敏逻辑。

## 核心改动

- 新增 `IHttpLogRedactor` 脱敏接口与 `DefaultHttpLogRedactor` 默认实现。
- 默认内容脱敏优先按 JSON 结构递归处理，未截断时保持 JSON 可解析。
- `LoggingHandler` 改为通过脱敏接口处理请求头、请求体和响应体。
- 保留 `EnableLogRedaction` 总开关；关闭时不会调用脱敏器。
- 移除响应内容审计中的 `Regex.Unescape`，避免破坏 JSON 转义结构。
