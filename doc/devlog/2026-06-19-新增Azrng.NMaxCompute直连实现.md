# Azrng.NMaxCompute 直连实现（去 Python 中转，S0 MVP）

## 本次目标

去掉 `Azrng.NMaxCompute`（v1.0.1）原本推荐的 Python 中转层，由 C# 直接调用阿里云 MaxCompute REST API 完成 SQL 提交、状态轮询、结果拉取，对外保留 ADO.NET Provider 形态。

最终架构决定：**单包方案**，所有代码合并入 `Azrng.NMaxCompute`，不再拆分 `Azrng.NMaxCompute.Direct` 独立包。版本号跳至 `2.0.0`（破坏性）。

本次交付为 S0 MVP：覆盖 V1/V4 签名、REST 客户端（重试 + 错误解析 + V4→V1 降级）、SQL 执行链路、Result API 取数（CSV，1 万行限制）；后续阶段（S1+）再补 Tunnel、复杂类型、STS、压缩。

## 核心改动

### 签名与账号（`Accounts/`）

- `IAccount` / `CloudAccount`：默认 V4，失败降级 V1
- `ISigner` / `V1Signer`（HMAC-SHA1 + Base64）/ `V4Signer`（4 层 HMAC-SHA256 派生 + HMAC-SHA1 终签，关键点：`k_secret = "aliyun_v4" + sk` 为字符串拼接而非 HMAC）
- `CanonicalStringBuilder`：精确复刻 PyODPS `_build_canonical_str`，含 query 排序、空值省略 `=`、`x-odps-*` 头以 `key:value` 输出、`Date` 缺失自动写回 GMT

### REST 客户端（`Rest/`）

- `OdpsRequest` / `OdpsResponse` / `OdpsError`（XML → JSON → ODPS-xxxxx 文本前缀三级回退解析）/ `OdpsException`（带 StatusCode/Code/RequestId/HostId/InstanceId）
- `OdpsRestClient`：4 次重试、指数退避、`GET/HEAD/DELETE` + `502/503/504`、V4→V1 自动降级（按 `need ak v3 support` / `accesskey acl denied` / `ODPS-0410051` / `invalid or missing` 判定）

### 核心执行链路（`Core/`）

- `Odps`：顶层入口，`RunSqlAsync` 提交 Job XML 解析 `Location` 拿 instance_id，`ExecuteSqlAsync` = run + wait + get result
- `Instance`：状态轮询，1s 起步指数退避封顶 30s，超时抛 `WaitTimeoutException`
- `JobXmlBuilder`：构造 `<Instance><Job><Tasks><SQL><Query><![CDATA[...]]></Query><Config>...</Config></SQL></Tasks></Job></Instance>`，settings JSON 注入 hints
- `SqlHints`：注入到 settings 的字典子类
- `InstanceResultParser`：解析 `<Tasks><Task><Result Format="...">...</Result></Task></Tasks>`
- `CsvResultParser`：RFC 4180 风格 CSV 解析（带引号转义、CRLF 兼容），S0 阶段所有列按字符串返回

### Provider（`Provider/`）

- `DirectOdpsQueryExecutor`：实现 `IQueryExecutor`，桥接 ADO.NET 抽象层与直连客户端

### Config 与连接字符串（破坏性重命名）

- `MaxComputeConfig`：
  - 删除：`ServerUrl` / `SecretKey` / `JdbcUrl`
  - 新增：`Endpoint` / `SecretAccessKey` / `Schema` / `Region` / `SecurityToken` / `TunnelEndpoint` / `UseV4Signature` / `Hints`
  - `Project` 改为必需，`MaxRows` 默认改 10000
- `MaxComputeConnectionFactory`：删除参数构造，仅保留 `MaxComputeConfig` / 连接字符串两种入口
- `MaxComputeConnectionStringBuilder`：完整重写，支持新字段并保留 `Url` / `SecretKey` 兼容读
- `MaxComputeConnection`：`DataSource` 改为 `_config.Endpoint`，`ParseConnectionString` 兼容旧键名

## 风险或遗留项

- **S0 范围限制**：Result API 路径 1 万行硬限制 + 所有列按字符串返回（Schema 列名正确，类型识别延后到 S1）。
- **未做集成测试**：本次未连接真实阿里云 MaxCompute 集群验证签名与 SQL 提交，原因：无可用 AK/SK 与项目。建议下一步在真实集群跑 `SELECT 1 AS a, 2 AS b` 完成端到端验证。
- **签名 Golden Test 待补**：当前测试覆盖格式正确性与确定性，但未对照 PyODPS 真实输出做字节级匹配。若线上首次出现 401，需要先用 PyODPS 跑固定输入拿到 `canonical_str` 与 `Authorization`，对照本实现修偏差。
- **`ARCHITECTURE.md` 未更新**：旧文档仍描述接口与实现分离的设计，需要后续单独更新或删除。
- **后续阶段（S1+）**：Instance Tunnel 解除 1 万行限制、复杂类型（ARRAY/MAP/STRUCT）、STS Token、压缩传输。

## 后续建议

- S1：实现 `ProtobufWireReader` + `Checksum`（CRC32+CRC32C）+ `InstanceDownloadSession`，覆盖 6 种基本类型 decoder（BIGINT/DOUBLE/STRING/BOOLEAN/DATETIME/DECIMAL）。
- S2：补 TIMESTAMP/DATE/JSON + ARRAY/MAP/STRUCT 递归 decoder。
- S3：`MaxComputeCommand.Hints` 属性注入 + Dapper 端到端读强类型 POCO。
- S4：`StsAccount` + zlib raw deflate + Tunnel→Result API 自动回退。
