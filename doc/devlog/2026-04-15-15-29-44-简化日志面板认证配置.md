# 简化日志面板认证配置

## 本次目标

- 移除 `DevLogDashboard` 的 `AuthorizationFilter`
- 将 `BasicAuthentication` 改为库内直接配置用户名密码
- 保持未配置认证时允许匿名访问

## 核心改动

- 删除 `AuthorizationFilter` 扩展点，统一保留内置 Basic 认证配置
- `BasicAuthentication` 新增 `UserName`、`Password`、`Realm`，无需宿主额外注册认证服务
- 中间件改为直接解析 Basic 请求头并校验用户名密码，成功后写入 `HttpContext.User`
- 服务注册阶段会校验 `BasicAuthentication.UserName` 和 `Password` 不能为空
- 更新 README 与中间件测试用例

## 风险或遗留项

- 这是一次 API 收敛，依赖 `AuthorizationFilter` 的宿主需要改为 `BasicAuthentication`
- 当前 Basic 认证为单账号密码配置，若后续需要多账号或更复杂授权，需要再扩展模型
