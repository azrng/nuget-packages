# 修复 SQL 解析器构建警告

## 本次目标
- 核查并修复 `JSqlParser.Net` 类库及测试项目构建时出现的可低风险处理警告。

## 核心改动
- 修复 ANTLR 语法中的重复 token、缺失 token 与可空匹配规则警告。
- 将 `ASTNodeAccessImpl.AppendTo` 调整为可重写方法，并补齐派生类型 `override`。
- 修复少量 nullable 警告，避免解析结果或表达式字符串潜在空值造成静态分析噪音。
- 对 SQL 领域公共 API 名称触发的 `CA1716` 做项目级抑制，避免重命名 `Select`、`Alias`、`Function` 等公共类型造成兼容性破坏。
- 修复测试项目中的 nullable 与 xUnit 集合数量断言警告。
