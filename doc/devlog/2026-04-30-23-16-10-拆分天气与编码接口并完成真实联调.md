# 拆分天气与编码接口并完成真实联调

## 本次目标
- 将 `INmcWeatherClient` 收缩为只负责按 `stationid/code` 查询天气
- 将省份、城市、编码解析能力拆到独立接口
- 对真实 `nmc.cn` 接口完成一次联调验证

## 核心改动
- 新增 `INmcLocationClient` / `NmcLocationClient`
  - 承担省份、城市、编码、映射等辅助查询职责
- 新增 `INmcWeatherQueryClient` / `NmcWeatherQueryClient`
  - 面向业务提供按城市名、城市名或编码的便捷天气查询
- 收缩 `INmcWeatherClient` / `NmcWeatherClient`
  - 仅保留 `GetWeatherByCityCodeAsync`
- 调整 `AddNmcWeather`
  - 现在会同时注册 `INmcLocationClient`、`INmcWeatherClient`、`INmcWeatherQueryClient`

## 真实联调发现与修复
- 线上最新城市站点编码并非旧资料中的纯数字编码，而是区分大小写的混合字符串，例如：`Wqsps`
- 原实现会把城市编码统一转大写，导致真实天气查询失败
- 现已修复：
  - 省份编码仍统一大写
  - 城市站点编码保留原始大小写
  - `GetCityAsync` 对字母数字形式的站点编码也会优先走 code 分支

## 测试与校验
- 单元测试已执行：`dotnet test test/Azrng.NmcWeather.Test/Azrng.NmcWeather.Test.csproj`
- 结果：`34 / 34` 通过
- 真实联调已执行：基于临时控制台程序调用线上 `http://www.nmc.cn`
- 联调结果：
  - `province=北京市 (ABJ)`
  - `city=北京 (Wqsps)`
  - `weather_by_code_station=Wqsps`
  - `weather_by_code_temp=18.6`
  - `weather_by_code_info=晴`
  - `weather_by_query_station=Wqsps`
