# 修复 Common.HttpClients.Next 代码审查问题

## 本次目标
- 完成 `Common.HttpClients.Next` 3.0.0 发版前的代码审查修复，覆盖必须修复 / 建议修改 / 可选项共 19 项；
- 跳过 #3（HttpHelperExtensions 文档与代码不一致）与 #6（PackageId 与目录名不一致）由用户决定保持不动；
- 新增 `Common.HttpClients.Next.Test` 测试项目，对齐 `Common.HttpClients.Test` 风格补齐测试用例。

## 核心改动

### 必须修复
- **#1 命名 options 不生效**：`LoggingHandler` 改注入 `IOptionsMonitor<HttpClientOptions>`，构造接受 `clientName`；`DefaultHttpLogRedactor` 改为 `internal sealed`，构造接受 `HttpClientOptions` 快照，不再注入 `IOptions<>`；`ServiceCollectionExtensions` 使用 `ActivatorUtilities.CreateInstance<LoggingHandler>(sp, name)`，按客户端名注册。
- **#2 Fallback NRE**：失败状态码路径上 `args.Outcome.Exception` 可能为 null，改为通过 `WrapFailedResponseAsException` 包装；同时把 Fallback 的 `ShouldHandle` 收窄为仅异常路径（HTTP 5xx 是真实响应，由调用方按 `StatusCode` 判断）。
- **#4 QueryStringBuilder**：补齐 `IDictionary<string, string>`（非 nullable）分支；`DateTimeFormat` 注释改为"内部使用"。

### 建议修改
- **#5**：`ResponseStream.DisposeAsync` 改用 `await _response.DisposeAsync()`。
- **#7 Polly 策略顺序**：调整为外 → 内：Fallback → Timeout → ConcurrencyLimiter → CircuitBreaker → Retry。`Timeout` 作为总耗时上限。
- **#8 configure 重复执行**：改用 `services.AddOptions<HttpClientOptions>(name).Configure(configure)`，configure 由 OptionsFactory 单次触发。
- **#9 二进制检测**：`IsBinaryMediaType` 按 MediaType 前缀和扩展名判断（image/ video/ audio/ octet-stream/pdf/zip/gzip/rar/7z/tar）。
- **#10 catch 静默吞异常**：所有 `catch` 改为 `_logger.LogWarning(ex, ...)`。
- **#11 null 校验**：`HttpHelperExtensions.CreateBearerHeaders` 增加 token 空值校验。
- **#12 UserAgent / DefaultHeaders**：移到 `ConfigureHttpClient` 在 client 层一次性设置，`HttpClientHelper.CreateRequestMessage` 不再每次重设。
- **#13 IValidateOptions**：新增 `HttpClientOptionsValidator : IValidateOptions<HttpClientOptions>`，reload 时同样生效。
- **#14 DateTimeFormat**：注释改为"内部使用"，避免与 README 描述冲突。
- **#15 IsFallbackResponse 魔法字符串**：抽常量到 `HttpClientHeaderNames.FallbackResponse`。

### 可选 / 仅参考
- **#16 LogRequestStart try/catch**：简化为单次 `LogInformation`。
- **#17 ShouldSkipLogging**：保留原逻辑，性能可接受。
- **#18 ConcurrencyLimit = 0**：验证范围改为 0..10000；0 表示禁用 limiter。
- **#19 Convert.ChangeType**：`HttpResult<T>.Success` 简化为强转。
- **#21 README PATCH**：修复重复 PATCH 示例。

### 关键设计决策
- **ConcurrencyLimiter 允许排队**：`AddConcurrencyLimiter(permitLimit, queueLimit)` 默认 `queueLimit=0` 会直接拒绝超限请求；改为 `queueLimit = Math.Max(permitLimit * 10, 100)`，由外层 `Timeout` 兜底排队等待时间。
- **MaxRetryAttempts = 0**：Polly v8 要求 `>=1`，0 时跳过 retry 策略添加。
- **DefaultHttpLogRedactor**：Regex 通过静态 `ConcurrentDictionary<string, Regex>` 缓存按敏感字段 pattern 复用。
- **RedactHeaders**：直接构造 case-insensitive 字典，避免大小写不同的同名 header 触发 `Dictionary` 构造的 `AddRange` 异常。

## 风险或遗留项
- **#3 / #6 保持不动**：`HttpHelperExtensions` 便利重载文档与代码不一致、`PackageId=Common.HttpClients` 与目录名不一致 —— 按用户决定保留。
- **IHttpLogRedactor 注册方式**：3.0.0 未发布，用户自定义 redactor 通过 `services.AddSingleton<IHttpLogRedactor, CustomRedactor>()` 注入仍可工作；`DefaultHttpLogRedactor` 改为 internal 后不再可由用户直接注册。
- **Polly 顺序变化**：原 Retry 在最外，新方案 Retry 在最内并由 `Timeout` 监督 —— 用户的总耗时上限 = `Timeout`。
- **Fallback 行为收紧**：原 5xx 会触发 Fallback；新方案仅在异常路径触发。如需对 5xx 也兜底，调用方需自行通过 `IsSuccess` / `StatusCode` 判断。
