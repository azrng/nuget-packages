# 修复 Common.Security 文档输出冲突

## 本次目标
- 修复 `Common.Security` 多目标框架构建时 XML 文档输出写入同一文件导致的 `CS0016` 错误。

## 核心改动
- 将 `src/Shared/Common.Security/Common.Security.csproj` 中的 `DocumentationFile` 改为 `$(OutputPath)$(AssemblyName).$(TargetFramework).xml`。
- 让 `netstandard2.1`、`net6.0`、`net7.0`、`net8.0`、`net9.0`、`net10.0` 分别生成独立 XML 文档文件，避免并行构建抢占同一路径。

## 修改文件
- `TASK.md`
- `src/Shared/Common.Security/Common.Security.csproj`
- `doc/devlog/2026-04-15-17-15-33-修复安全组件文档输出冲突.md`

## 校验情况
- 已执行：`dotnet build src\\Shared\\Common.Security\\Common.Security.csproj -c Release -p:GenerateDocumentationFile=true`
- 已确认：`bin/Release/<TargetFramework>/` 下生成了独立 XML 文件，项目根目录不存在 `Common.Security.xml`
- 构建结果：成功，存在已有 `SYSLIB0021`、`SYSLIB0053` 警告，本次未处理

## 风险或遗留项
- 本次仅修复 `Common.Security` 的 XML 输出冲突；仓库内其他多目标框架项目若在特定 CI 配置下也存在相同问题，需要按同样方式单独排查。
