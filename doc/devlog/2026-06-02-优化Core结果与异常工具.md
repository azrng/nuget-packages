# 优化 Core 结果与异常工具

## 本次目标
- 根据 `Azrng.Core` 结果包装与异常体系抽离建议，补充公共包内通用结果创建、数据读取和异常转换能力。

## 核心改动
- `ResultModel.Success()` 与 `ResultModel<T>.Success()` 增加自定义成功消息重载，保留业务消息。
- 新增 `ResultModelFactory`，统一创建成功、失败和异常转换结果。
- 新增 `ResultModelExtensions`，提供 `DataOrEmpty`、`DataOrDefault` 等数据安全读取扩展。
- 新增 `ResultModelExceptionExtensions`，提供 `Exception.ToFailureResult<T>()`。
- 补充 `Azrng.Core.Test` 单元测试覆盖消息保真、异常错误码复用和数据读取扩展。

## 风险或遗留项
- 普通异常默认使用 `Exception.Message` 生成失败结果；服务端 API 如需隐藏内部细节，应传入 `message` 覆盖。
- 本次只上移公共能力，未同步替换外部应用中的 `DatabaseResultModel` 调用。
