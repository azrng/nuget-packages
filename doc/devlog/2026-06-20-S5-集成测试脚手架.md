## 背景

S0-S4 已通过 fixture 单元测试覆盖全部可验证逻辑。S5 剩余两块——PyODPS 对照基准与真实集群端到端——都依赖真实 MaxCompute 集群，无法在 CI 验证。本次交付一个 env-gated 集成测试脚手架，让用户填入凭据即可一键联调，把"做完"推到仅剩真实集群验证。

## 完成范围

### 集成测试脚手架

- `Integration/MaxComputeIntegrationTest.cs`：
  - `MaxComputeIntegrationTestBase`：`TryLoadConfig` 从环境变量读取配置；`LoadConfigOrSkip` 返回 null 时测试无操作跳过
  - 环境变量：`MAXCOMPUTE_TEST_ENDPOINT/ACCESS_ID/SECRET_KEY/PROJECT/REGION`（必需），`SCHEMA/TUNNEL_ENDPOINT/SECURITY_TOKEN`（可选）
  - 三个测试类：
    - `ConnectionSmokeTest.SelectOne_ReturnsRow`：`SELECT 1` 烟测
    - `ConnectionSmokeTest.SelectMixedTypes_TunnelPath_TypesResolved`：BIGINT/DOUBLE/STRING/BOOLEAN/DECIMAL 混合，验证 Tunnel 路径 + `ColumnTypes` 解析
    - `LargeResultSetTest.Select_OverTenThousandRows_AllReturned`：超 1 万行验证 Tunnel 不截断（Result API 会截断到 10000）
  - `[Trait("Category","Integration")]`：`dotnet test --filter "Category=Integration"` 可单独跑
- `Integration/SimpleHttpClientFactory.cs`：测试用 `IHttpClientFactory`（共享 HttpClient）

### xUnit v2 skip 说明

xUnit v2 无原生 `Assert.Skip`（v3 才有），故未配置环境变量时测试以「无操作通过」形式跳过——CI 保持绿色，填入凭据后断言才真正执行。

## 未做（明确推迟到真实集群联调）

- **zlib raw deflate 压缩传输**：当前不主动请求压缩，多数场景可跑通；真实集群若强制压缩才会暴露
- **PyODPS 对照基准测试套件**：需真实集群跑固定 SQL 矩阵，与 C# 结果递归深度比对
- **TIMESTAMP 秒/纳秒单位、复合类型 CRC 语义的集群级验证**：fixture 已验证逻辑，真实数据流待联调确认
