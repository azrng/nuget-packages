# Azrng.NMaxCompute S1：Instance Tunnel + 基础类型 decoder

日期：2026-06-20
任务 ID：T017
关联：S1 阶段（plan: `azrng-nmaxcompute-python-python-c-pure-catmull.md`）

## 背景

S0 已打通 `Azrng.NMaxCompute` 直连 ODPS REST → Result API 的端到端链路（CSV/文本，10000 行限制）。
S1 目标是把 Instance Tunnel（protobuf wire 流式协议、CRC32C 校验）和 6 种基础类型 decoder
接入数据读取链路，解除行数限制、提供真实类型返回。

## 完成范围

### S1-1 Checksum（CRC32 + CRC32C）

- `Tunnel/Wire/Crc32.cs`：标准 CRC-32（ISO-HDLC，多项式 `0xEDB88320`），表驱动，等价 `zlib.crc32`
- `Tunnel/Wire/Crc32C.cs`：CRC-32C（Castagnoli，多项式 `0x82F63B78`）
- `Tunnel/Wire/Checksum.cs`：字段级 checksum 包装器（默认 CRC32C），暴露
  `UpdateBool / UpdateInt / UpdateLong / UpdateFloat / UpdateDouble / Update(ReadOnlySpan<byte>)`

测试覆盖：RFC 3720 / zlib 标准向量、chunked update 等价性、reset 行为、各字段类型 LE 编码。

### S1-2 ProtobufWireReader

- `Tunnel/Wire/ProtobufWireReader.cs`：手写 wire 解码器
  - `ReadVarUInt32 / ReadVarUInt64`
  - `ReadSInt32 / ReadSInt64`（zigzag 解码）
  - `ReadFixed32 / ReadFixed64`、`ReadSFixed32 / ReadSFixed64`
  - `ReadFloat / ReadDouble / ReadBool`
  - `ReadBytes / ReadString`（length-delimited）
  - `ReadTag` → `(FieldNumber, WireType)`
- `Tunnel/Wire/TunnelWireConstants.cs`：tunnel 魔数 + wire type 常量
  - `TunnelMetaCount = 33554430`
  - `TunnelMetaChecksum = 33554431`
  - `TunnelEndRecord = 33553408`
  - `TunnelEndMetrics = 33554176`

测试覆盖：单/多字节 varint、zigzag 映射、LE fixed/double/float、tag 拆分、流末尾截断抛错。

### S1-3 InstanceDownloadSession

- `Tunnel/TableSchema.cs`：简化版 schema（列名 + 类型字符串），从 JSON 解析
- `Tunnel/InstanceDownloadSession.cs`：对应 PyODPS `odps/tunnel/instancetunnel.py::InstanceDownloadSession`
  - `CreateAsync`：`POST /projects/<proj>/instances/<id>?downloads`，注入 `x-odps-tunnel-version: 6`
  - `ReloadAsync`：`GET ...?downloadid=<id>`
  - 响应 JSON 解析：`DownloadID / Status / RecordCount / Schema / QuotaName`
  - 状态枚举：Normal / Closed / Expired / Failed / Initiating / Unknown

测试覆盖：状态字符串解析、JSON 解析、空 body / 非法 JSON 抛错、字段缺失时保留默认值。

### S1-4 基础类型 decoder

- `Tunnel/Types/OdpsType.cs`：基础类型枚举
- `Tunnel/Types/ITypeDecoder.cs`：解码器接口，输入 `ProtobufWireReader + Checksum`，输出 `object?`
- `Tunnel/Types/BasicDecoders.cs`：6 种基础类型实现
  - `IntegerDecoder`：`ReadSInt64 + UpdateLong`（覆盖 tinyint/smallint/int/bigint）
  - `FloatDecoder` / `DoubleDecoder`：fixed32/64 + UpdateFloat/UpdateDouble
  - `BooleanDecoder`：单字节 varint + UpdateBool
  - `StringDecoder`：length-delimited bytes + Update(bytes)（覆盖 string/binary/varchar/char/json）
  - `DecimalDecoder`：length-delimited UTF-8 + Update(bytes)，按 invariant culture 解析 decimal
- `Tunnel/Types/DateTimeDecoder.cs`：DATETIME/DATE decoder
  - DATETIME：ms 时间戳 → 本地时区 DateTime（与 PyODPS `local_timezone=True` 默认一致）
  - DATE：自 epoch 起的天数 → `DateOnly`
- `Tunnel/Types/TypeDecoderFactory.cs`：类型字符串 → decoder 查表
  - 支持 `decimal(p,s)` 自动归一化

测试覆盖：每种类型 wire 字节解码、CRC 累计与原生 Checksum 等价、factory 路由（含大小写、decimal 精度参数）、不支持类型抛 `NotSupportedException`。

### S1-5 DataReader 接 Tunnel

- `Tunnel/TunnelRecordReader.cs`：单条记录解码器
  - 循环读 tag → field_index，处理 TUNNEL_END_RECORD（CRC32C 验证）、META_COUNT/CHECKSUM/END_METRICS、普通字段
  - 流末尾优雅返回 null（仅在「整条记录尚未开始」时把 InvalidDataException 视为正常结束）
- `Tunnel/TunnelResultMaterializer.cs`：`TunnelRecordReader → QueryResult`，把 NULL 转为 DBNull
- `Models/QueryResult.cs`：新增 `ColumnTypes`，与 `Columns` 等长
- `MaxComputeDataReader.cs`：
  - 构造时根据 `ColumnTypes` 推断每列 CLR 类型
  - `GetFieldType` 返回真实 CLR 类型（缺失时回退到 string，保持 S0 兼容）
  - `GetDataTypeName` 优先返回 ODPS 类型字符串

测试覆盖：
- TunnelRecordReader：单条记录混合类型、多条记录、CRC 校验失败抛 `OdpsException`、double 列、字段缺失留 null
- TunnelResultMaterializer：多行物化、列名/类型透传
- DataReaderTypeMapping：所有基础 ODPS 类型 → CLR 类型映射、ColumnTypes 缺失时回退 string

## 测试结果

`dotnet test` 全绿：136 个测试通过。

## 当前缺口（待 S2-S5 阶段处理）

- Tunnel 流的实际 HTTP 拉取（`OpenDataReaderAsync`）尚未接入 `InstanceDownloadSession`，
  需要在 `OdpsRestClient` 上增加 streaming read API。S1 阶段聚焦 wire 解码正确性，
  并通过 fixture 字节验证。
- TIMESTAMP / DATE 复杂语义（时区、纳秒精度）暂未实现
- ARRAY / MAP / STRUCT 递归 decoder 留待 S2
- STS / 压缩 / Tunnel→Result API 自动回退留待 S4

## 关键设计点

- 单包架构：所有代码在 `Azrng.NMaxCompute` 一个包内，没有拆出 `.Direct`（合并回主包后保持）
- 版本号：仍维持 1.0.1，不随 S1 改动升级（用户要求）
- 测试驱动：fixture 字节直接构造 wire 流，校验 CRC 累积、解码值、与原生 Checksum 一致性
- InternalsVisibleTo：仅 `Azrng.NMaxCompute.Test` 可见 internal 构造器，便于无网络环境测试
