# 修复雪花ID短值问题

## 本次目标

修复 `Azrng.Core.Helpers.Snowflake.NewId()` 返回类似 `3809281` 这类不包含时间戳高位的非标准雪花 ID 风险。

## 核心改动

- 为雪花 ID 的机器位、序列位、时间戳位移和掩码补充显式常量，避免依赖难读的内联位运算表达式。
- 统一通过 `CreateId` 拼装 ID，确保返回值包含 `41bit` 时间戳、`10bit` 机器 ID 和 `12bit` 序列号。
- 为初始化和 ID 生成过程补充锁保护，避免并发首次初始化时状态竞争。
- 将公开 `Sequence` 属性映射到实际内部序列字段，避免属性设置无效。
- 补充单测，覆盖生成 ID 必须包含时间戳高位、指定时间生成后可解析回原时间。
- 将 `Snowflake` 收敛为静态工具类，仅公开 `NewId()`、`NewId(DateTime)` 和 `TryParse(...)`。
- 移除公开 `Sequence` 和 `GetId(DateTime)`，避免外部修改内部序列或误用时间边界 ID。
- 为 `NewId(DateTime)` 增加起始时间与 41 位时间戳范围校验，为 `TryParse(...)` 增加非法 ID 校验。
